================================================================================
                          PROGRESS STORAGE
================================================================================

Local Storage Key: course_progress_{userId}_{courseId}

Example:
  course_progress_user123_math456: {
    "courseId": "math456",
    "userId": "user123",
    "completedLessons": ["lesson1", "lesson2"],
    "currentLessonId": "lesson2",
    "progress": 100,
    "lastUpdated": "2024-01-15T10:30:00Z"
  }

Backend Sync (Optional):
  POST /api/progress
  - courseId
  - completedLessons
  - currentLessonId
  - progress
  - auth token required

================================================================================
                         KNOWN LIMITATIONS
================================================================================

- No quiz completion tracking
- Backend sync is optional (localStorage works standalone)
- No multi-device sync
- No detailed analytics

Future enhancements:
